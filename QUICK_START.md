# دليل الاستخدام السريع - نظام إدارة الأصول

## ⚡ البدء السريع (5 دقائق)

### 1. إعداد GitHub للمرة الأولى
```
1. اذهب إلى: https://github.com/settings/tokens
2. اضغط: "Generate new token (classic)"
3. اختر صلاحيات: ☑ repo
4. انسخ الرمز
```

### 2. إعداد النظام
```
1. افتح dashboard.html
2. اضغط: "إعدادات GitHub"
3. الصق رمز الوصول
4. فعل: ☑ تفعيل الحفظ التلقائي
5. اضغط: "حفظ الإعدادات"
```

### 3. الاستخدام اليومي
```
✅ أضف أصول جديدة
✅ عدل الأصول الموجودة
✅ احذف الأصول غير المرغوبة
✅ اضغط "حفظ التغييرات"
✅ البيانات تُحفظ تلقائياً على GitHub!
```

## 🎯 الميزات الرئيسية

| الميزة | كيفية الاستخدام |
|--------|-----------------|
| **إضافة أصل** | اضغط "إضافة جهاز جديد" |
| **تعديل أصل** | اضغط على الصف في الجدول |
| **حذف أصل** | حدد واضغط "حذف" |
| **البحث** | اكتب في شريط البحث |
| **الفلترة** | اختر من القوائم المنسدلة |
| **الحفظ** | اضغط "حفظ التغييرات" |
| **الحفظ على GitHub** | اضغط "حفظ على GitHub" |
| **التصدير** | Excel أو CSV |

## 🔧 إعدادات GitHub التفصيلية

### الحقول المطلوبة:
- **رمز الوصول**: من GitHub Personal Access Token
- **اسم المستخدم**: اسم حسابك على GitHub
- **اسم المستودع**: اسم المستودع الذي يحتوي على `assets.json`
- **الحفظ التلقائي**: فعله للحفظ التلقائي عند حفظ البيانات

### مثال:
```
رمز الوصول: ghp_xxxxxxxxxxxxxxxxxxxx
اسم المستخدم: AhmedIbrahimMoustafa
اسم المستودع: samcrete-assets
```

## 🚨 حل المشاكل الشائعة

### ❌ "Bad credentials"
**الحل**: تأكد من صحة رمز الوصول وأنه لم ينتهِ

### ❌ "Repository not found"
**الحل**: تأكد من اسم المستخدم واسم المستودع

### ❌ البيانات لا تظهر
**الحل**: تأكد من وجود ملف `assets.json` في المستودع

### ❌ التغييرات لا تحفظ
**الحل**: فعل "الحفظ التلقائي" في إعدادات GitHub

## 📱 نصائح للاستخدام الأمثل

- **احفظ بانتظام**: اضغط "حفظ التغييرات" بعد كل تعديل مهم
- **استخدم البحث**: للعثور على الأصول بسرعة
- **راجع الإعدادات**: إذا واجهت مشاكل في GitHub
- **استخدم التصدير**: للحصول على نسخة احتياطية
